Study Specific Metadata

Some metadata columns may not be relevant to a given study. If the metadata doesn't exist for an optional column it should be filled in with 'NA' not 'na', 'n.a.', 'N/A' or any other Not Available indicator. DO NOT delete the columns from the metadata template.

If metadata exists that does not fit into any of the default columns, additional columns may be added. The format should be as follows:

For Example:

  • 1: AdditionalMetaData
  • 2: (Name of the metadata column)
  • 3: Optional
  • 4: (Type of the metadata: Text, Number, Date, Time, etc)
  • 5: (Unit of the metadata: If applicable)

This optional metadata is stored separately from the default columns so analysis based on these columns may be limited.


Term Definitions

Certain terms in our metadata may be used differently in others contexts. For reference we define them as follows:

  • Intervention: Any procedure, medication, or other type of treatment performed on a subject.
  • Specimen: The initial material collected prior to any analysis.
  • Aliquot: Any sub-divisions of the initial specimen.
  • Sample: The result of processing an aliquot through some type of wet lab procedure.
  • RawData: The result of sequencing a sample. Typically this is the stage we will receive the data and accompanying metadata.
  • Result: The final result of running a computational tool (such as Qiime) on RawData. This will typically be generated by MMEDS and so it may be left blank.

Metadata field entries should not contain special characters '$', '*', ';', etc. Metadata files containing entries with these characters cannot be uploaded to the server. All text fields will be converted to lowercase when stored in the database, so make sure none of the entries are case sensitive.


Protocol vs. Protocols

As you may have noticed there are several columns with similar names. Such as SampleProtocol and SampleProtocols, SpecimenProtocol and SpecimenProtocols, etc. These refer to the methods used to produce a particular Specimen, Aliquot, etc. For example, a particular Sample, the 'SampleProtocols' columns in the same row refer to the standard procedure, such as FACS sorting, used to create this Sample from the associated Aliquot. SampleProtocol contains information related to that particular run of FACS sorting. This includes, when was it performed, who performed it and if there was anything unusual about the sorting in this run. Similarly RawDataProtocols refer to general methods for processing Samples into RawData, such as Illumina MiSeq. For example, RawDataProtocol refers to a particular instance of running Illumina MiSeq.


Column Specifics

There are certain fields that will only accept values that fall into certain categories:

RawDataID:

  • This is the unique identifier for each sample that has already gone through sequencing.
  • Equivalent to #SampleID when using qiime
  • If the reads are demultiplexed the file names should follow the format
  • (RawDataID)_(AdditionalInformation)_R1_001.fastq.gz
    E.G.
    Forward Read: S33_L001_R1_001.fastq.gz
    Reverse Read: S33_L001_R2_001.fastq.gz

SpecimenBodySite & SpecimenType:

  • Values should be taken from UBERON https://www.ebi.ac.uk/ols/ontologies/uberon , using the terms as provided on the website
  • The UBERON ID should be included in their respective UberonCode columns

Ethnicity

Categories taken from NIH standards. https://grants.nih.gov/grants/guide/notice-files/not-od-15-089.html

  • American Indian or Alaska Native
  • Asian
  • Black or African American
  • Hispanic or Latino
  • Native Hawaiian or Other Pacific Islander
  • White

Note: One subject may have multiple ethnicities. Include each in a separate row. It is acceptable to duplicate other information for this purpose.

Sex

  • Male
  • Female
  • Transgender Male
  • Transgender Female

Note: For someone to qualify as Transgender they must have undergone some procedure or received some medication such as hormone therapy. Please include this information under 'Intervention' if known.

ICDCode

  • Illnesses should be classified by ICD 10 code and recorded in this column.
  • If the subject is known to be healthy use the code XXX.XXXX
  • If the subject's health status is unknown enter "NA"
  • All codes must contain all seven digits + the decimal. Fill empty digits with 'X'. E.G. K55.5 should become K55.5XXX
  • If only part of the code is known fill in the rest with 'X' E.G. K45 should become K45.XXXX
  • If the subject is an animal, use the code for a human subject that most closely resembles its condition.

Intervention

InterventionTypes include:

  • Medication
  • Surgery
  • Physical Therapy
  • Environmental
  • Other
  • Treatments should include the NCD code if it is a drug or CPT code for a surgical procedure.
  • If the treatment does not have a code from this list, such as a dietary intervention, then the intervention should be described well enough to be reproducible.
